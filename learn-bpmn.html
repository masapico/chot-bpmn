BPMNとは？

BPMN (Business Process Model and Notation) は、ビジネスプロセスを視覚的に表現するための標準記法です。業務の流れを明確にし、関係者間での共通理解を促進することを目的としています。

このガイドでは、BPMN図を作成する上で基本となる要素とその意味について説明します。

主要な要素

1. フローオブジェクト (Flow Objects)

プロセスの振る舞いを定義する主要なグラフィカル要素です。

  • イベント (Events) 意味: プロセスの中で「何かが起こる」ことを示します。プロセスの開始、終了、またはプロセス中に発生する事象を表します。
    • 開始イベント (Start Event): プロセスの開始点。通常は細い線の円で表現されます。
    • 中間イベント (Intermediate Event): プロセス中に発生するイベント。二重線の円で表現され、境界に配置されることもあります。
    • 終了イベント (End Event): プロセスの終了点。太い線の円で表現されます。

    例: 「顧客からの注文受付」（開始イベント）、「承認通知受信」（中間イベント）、「処理完了」（終了イベント）

    [イベントの記号のイメージ]
  • アクティビティ (Activities) 意味: プロセスの中で行われる「作業」や「仕事」を表します。
    • タスク (Task): それ以上分割できない基本的な作業単位。角丸の長方形で表現されます。様々な種類（ユーザータスク、サービスタスクなど）があります。
    • サブプロセス (Sub-Process): より詳細なプロセスフローを含む複合的なアクティビティ。角丸の長方形の中に「+」マーク（折り畳み時）や詳細なフローが描かれます。

    例: 「見積書作成」（タスク）、「承認プロセス」（サブプロセス）

    [アクティビティの記号のイメージ]
  • ゲートウェイ (Gateways) 意味: プロセスの流れを制御し、分岐させたり合流させたりするポイントを示します。ひし形で表現されます。
    • エクスクルーシブゲートウェイ (Exclusive Gateway): 条件に基づいて、複数の経路のうち1つだけを選択します。「X」マークで示されることが多いです。
    • パラレルゲートウェイ (Parallel Gateway): 複数の経路を同時に実行（分岐）、または複数の経路の完了を待ってから次に進みます（合流）。「+」マークで示されます。
    • インクルーシブゲートウェイ (Inclusive Gateway): 条件に基づいて、1つ以上の経路を選択します。「O」マークで示されます。

    例: 「金額が10万円以上か？」（エクスクルーシブゲートウェイでの分岐）、「書類Aと書類Bの両方が揃ったら次に進む」（パラレルゲートウェイでの合流）

    [ゲートウェイの記号のイメージ]

2. 接続オブジェクト (Connecting Objects)

フローオブジェクト同士を接続し、プロセスの順序や関係性を示します。

  • シーケンスフロー (Sequence Flow) 意味: プロセス内のアクティビティが実行される順序を示します。実線の矢印で表現されます。

    例: 「タスクA」→「タスクB」

    [シーケンスフローの記号のイメージ]
  • メッセージフロー (Message Flow) 意味: 異なるプール（参加者）間のメッセージのやり取りを示します。破線の矢印で、始点に白丸、終点に白抜きの矢印がつくことが多いです。

    例: 「顧客プール」から「自社プロセスプール」への「注文メッセージ」

    [メッセージフローの記号のイメージ]
  • 関連 (Association) 意味: フローオブジェクトと成果物（テキスト注釈やデータオブジェクトなど）を関連付けます。点線で表現されます。

    例: 「タスクA」と「補足説明のテキスト注釈」を関連付ける

    [関連の記号のイメージ]

3. スイムレーン (Swimlanes)

プロセス内のアクティビティの責任範囲や役割分担を明確にするための視覚的な区画です。

  • プール (Pool) 意味: プロセス全体の参加者（例: 企業、組織、システム）を表します。通常、大きな長方形で表現されます。

    例: 「顧客」、「営業部」、「経理部」

    [プールの記号のイメージ]
  • レーン (Lane) 意味: プール内の特定の役割や担当者を表します。プールを水平または垂直に分割する形で表現されます。

    例: 「営業部」プール内の「担当者レーン」と「上長レーン」

    [レーンの記号のイメージ]

4. データ (Data)

プロセスで使用される情報や成果物を表します。

  • データオブジェクト (Data Object) 意味: プロセス中で作成、使用、更新されるデータを示します。書類のような形のアイコンで表現されます。

    例: 「注文書」、「見積書」

    [データオブジェクトの記号のイメージ]
  • データストア (Data Store) 意味: プロセス外部に永続的に保存されるデータを示します。データベースの円筒形のようなアイコンで表現されます。

    例: 「顧客データベース」、「商品マスタ」

    [データストアの記号のイメージ]

5. 成果物 (Artifacts)

プロセス図に補足的な情報を加えるために使用されます。

  • テキスト注釈 (Text Annotation) 意味: プロセス図の特定の部分に説明やコメントを追加します。左側が開いた角括弧のような記号で示され、関連線でフローオブジェクトに接続されます。
    [テキスト注釈の記号のイメージ]
  • グループ (Group) 意味: 複数の要素を視覚的にグループ化します。分析や文書化の目的で使用され、プロセスの実行順序には影響しません。破線の角丸長方形で表現されます。
    [グループの記号のイメージ]

簡単なBPMN図の例

以下は、簡単な「休暇申請プロセス」の例です。

  1. 開始イベント: 「休暇申請」
  2. タスク: 「申請書作成」
  3. タスク: 「上長承認」
  4. エクスクルーシブゲートウェイ: 「承認結果は？」
    • 承認された場合 → タスク: 「人事部へ通知」 → 終了イベント: 「申請処理完了」
    • 却下された場合 → タスク: 「申請者へ通知」 → 終了イベント: 「申請処理完了」

このプロセスを、プール（例: 申請者、上長、人事部）とレーンを使って役割分担を明確に表現することができます。
